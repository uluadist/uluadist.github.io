AutoBuild Status

For each OS/arch combination, in order for a rock install to be attempted the following pre-conditions must be satisfied:

  • the version format of the rock must be semver compatible
  • the rockspec download must succeed
  • the OS must be supported by the rock
  • the rock must not depend on external C libraries (for now: WIP)
  • the version format of each listed rock dependency must be semver compatible
  • the rock must not depend on another rock whose build failed or which is not available

If all these conditions are satisfied, the rock install (i.e. luarocks install ...) is attempted and the stdout and stderr outputs are logged in case of failure.

Following a successful rock install for each OS/arch combination (for now: WIP), it is also verified that no path conflict arise. Rocks belonging to the same product are successfully bundled together and the resulting ULua package name will be of the form rootname-bundle. All other path conflict results in a failure.
